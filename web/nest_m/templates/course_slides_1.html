Getting started with Python

How to interact with Python like a software engineer

Mac OSX

On Mac OSX or Linux operating systems accessing Python is best done through the Terminal application.


Windows

Using Windows is slightly different, however you can similarly use the run application.

Read-Eval-Print-Loop, REPL

The intent of using Python from Terminal/Run, is so that we may interact with Python's REPL--often referred to as the Python Interpeter.

The Python Interpreter

The interpreter is an extremely valuable tool because it allows you test code as you write it, as well as interact with the software modules in your environment.

Using the Interpreter

The Python Standard Library

By default the Python language comes with a set of modules and packages immediately available in any environment.

These modules provide basic functionality and interfaces for interacting with different things.

For example,

'import os' provides basic functions for interacting with the operating system.

'import urllib'gives a basic set of tools for parsing and constructing web addresses.

The list of available tools in the Python Standard Library is very extensive and being familiar with its features is extremely valuable and can teach you the blueprint of how to write Python software.

The Python Standard Library By Example, by Doug Hellmann

The Python Package Index

Python is known for its feature of Fast Prototyping.

What that really means is the community has allowed Python to be highly extensible by providing a number of additional packages and software tools that you can instantly download and incorporate into your application.

When writing Python software, if you'd like to use a package or framework already available to the community, you simply have to install it, and it becomes available to you to import just like any standard library.

You can find an entire list of community packages available to download at pypi.python.org

Installing packages from PyPi can be done using the pip tool.

pip install package-name
